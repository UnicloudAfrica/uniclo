:::step{number=1 icon=Search title="Provider Discovery" navigation="Sidebar > Provider Discovery"}
Provider discovery lets you connect to and explore external cloud providers. You can sync users, projects, and resources from connected providers. Think of it as scouting new territories and mapping what is available.
:::

:::screenshot{caption="Your Provider Discovery page"}
:::

:::step{number=2 icon=ShieldCheck title="Serverless DR" navigation="Sidebar > Disaster Recovery > Serverless DR"}
Serverless Disaster Recovery keeps a standby copy of your important servers ready to go. If the original server goes down, the backup can take over. It is like having a spare tire that is always inflated and ready to use.
:::

:::step{number=3 icon=ArrowLeftRight title="Migrations" navigation="Sidebar > Disaster Recovery > Migrations"}
The migration tool helps move workloads from one location to another. Whether you are moving servers between regions or from an external provider, the migration wizard guides you through every step.
:::

:::screenshot{caption="Your Migrations list"}
:::

:::step{number=4 icon=Zap title="Protection" navigation="Sidebar > Disaster Recovery > Protection"}
Protection services provide automated backup policies for servers. Set up schedules, retention rules, and recovery points to ensure data is always safe.
:::

:::step{number=5 icon=Scaling title="Auto Scaling" navigation="Sidebar > Networking > Auto Scaling"}
Auto scaling automatically adds or removes servers based on demand. When traffic spikes, new servers spin up. When things calm down, they spin down. It is like a restaurant that magically adds tables when it gets busy.
:::

:::step{number=6 icon=Bot title="Infrastructure Agent" navigation="Sidebar > Infrastructure Agent"}
The infrastructure agent runs inside virtual machines to help manage and monitor them from the platform. It collects metrics, runs health checks, and enables advanced management features.
:::

:::callout{type=warning}
Advanced services may require additional configuration in your cloud provider accounts. Make sure regions and credentials are properly set up before using these features.
:::
